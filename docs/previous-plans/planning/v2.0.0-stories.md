## Dashboard (3 stories)

### US-001: KPI Dashboard View
**Priority:** High

As a billing operator, I want to see KPI widgets (Total AR, active meters, payments today, overdue count) on the dashboard so I can quickly assess system health.

**Acceptance Criteria:**
- Exactly 4 KPI cards rendered on load: Total AR, Active Meters, Payments Today, Overdue > 30d
- Each card shows a numeric value, % change vs. prior period, and a trend arrow
- KPIs refresh every 60 seconds via polling
- Clicking a KPI navigates to the relevant detail page
- Cards are responsive (2×2 grid on desktop, stack on mobile)

### US-002: Dashboard Area Filter
**Priority:** High

As a regional manager, I want to filter dashboard KPIs by area so I can isolate performance by location.

**Acceptance Criteria:**
- Dropdown filter at top of dashboard listing all active areas
- Selecting an area re-fetches KPIs scoped to that area's meters and customers
- "All Areas" option resets to global view
- Filter state persists across page navigation within session

### US-003: Dashboard Date Range
**Priority:** Med

As a billing operator, I want to set a custom date range for dashboard KPIs so I can compare period-over-period trends.

**Acceptance Criteria:**
- Date range picker with presets: Today, This Week, This Month, Last Month, Custom
- Custom range allows start date ≤ end date, max 365-day window
- All KPI values recalculate based on selected range
- Range is reflected in the URL query string for shareability

## Customers (4 stories)

### US-004: Customer Card Grid
**Priority:** High

As a billing operator, I want to view customers in a card grid showing name, account#, area, balance, and status so I can browse accounts visually.

**Acceptance Criteria:**
- Cards display in a responsive grid (4 cols desktop, 2 tablet, 1 mobile)
- Each card shows: customer name, account number, area badge, current balance, status pill (Active/Suspended/Closed)
- Cards sortable by name, balance, area
- Search box filters by name or account number with debounce (300 ms)
- Pagination with page size selector (12/24/48)

### US-005: Inline Customer Edit
**Priority:** High

As a billing operator, I want to edit customer details inline on the card or detail page so I can update records efficiently.

**Acceptance Criteria:**
- Clicking an edit icon on a card opens inline editable fields
- Editable fields: phone, email, address, unit type, status
- Save button writes changes via PATCH /api/customers/{id}
- Unsaved changes prompt confirmation before navigation
- Audit log entry created on every save

### US-006: Attach Documents to Customer
**Priority:** Med

As a billing operator, I want to attach documents (PDF, JPG, PNG) to a customer record so I can store signed agreements and correspondence.

**Acceptance Criteria:**
- Upload button on customer detail page accepts PDF, JPG, PNG (max 10 MB each)
- Drag-and-drop zone with visual feedback
- Attachments listed with file name, upload date, download link
- Max 20 attachments per customer
- Virus scan on upload; infected files rejected

### US-007: Customer Statement
**Priority:** Med

As a customer, I want to view/download my statement showing all transactions in a date range so I can reconcile my account.

**Acceptance Criteria:**
- Statement generated for any date range (max 12 months)
- Sections: opening balance, invoices, payments, adjustments, closing balance
- Download as PDF with company letterhead and QR code
- Statement URL is shareable and expires after 24 hours
- Email-to-customer option triggers async job

## Meters (4 stories)

### US-008: Meter Type Icons
**Priority:** Low

As a billing operator, I want meters displayed with type-specific icons (Water, Electric, Gas, Steam, Chilled Water) so I can identify meter types at a glance.

**Acceptance Criteria:**
- Icon set: faucet (Water), lightning (Electric), flame (Gas), gauge (Steam), snowflake (Chilled Water)
- Icons rendered on map pins, list rows, and card views
- Legend tooltip on hover showing meter type and model
- Color-coded border per type

### US-009: Meter Relay Signals
**Priority:** High

As a field technician, I want to send relay signals (Connect/Disconnect/Limit) to smart meters so I can control service remotely.

**Acceptance Criteria:**
- Action buttons: Connect, Disconnect, Limit (threshold in Amps/kL)
- Confirmation dialog with reason input (required for Disconnect)
- Signal queued and sent via AMQP; status tracked (Pending/Sent/Acknowledged/Failed)
- Real-time status update via WebSocket
- Failed signals auto-retry 3× with exponential backoff

### US-010: Meter Lifecycle Actions (11 Actions)
**Priority:** High

As a billing operator, I want to perform lifecycle actions on meters so I can manage installation to decommissioning.

**Acceptance Criteria:**
- Exactly 11 actions available based on meter status: Install, Inspect, Activate, Suspend, Resume, Replace, Calibrate, Transfer, Retrieve, Decommission, Retire
- Each action has role-based permission check
- Action history logged with timestamp, user, old/new status
- State machine enforces valid transitions only
- Bulk action support for selected meters

### US-011: Meter Lifecycle History
**Priority:** Med

As a billing operator, I want to view the full lifecycle timeline of a meter so I can audit its service history.

**Acceptance Criteria:**
- Timeline view sorted descending by date
- Each event shows: date, action type, user, notes
- Filter by action type
- Export timeline as CSV
- Events link to related work orders if applicable

## Balances (3 stories)

### US-012: Balance Summary Tabs (5 Tabs)
**Priority:** High

As a billing operator, I want to see 5 summary tabs on the Balances page so I can view outstanding, overdue, credit, disputed, and reconciled balances.

**Acceptance Criteria:**
- Tabs: Outstanding, Overdue, Credit, Disputed, Reconciled
- Each tab shows a table with customer, balance, aging bucket
- Totals row at bottom of each tab
- Drill-down click opens per-customer detail
- Tab state persisted in URL hash

### US-013: Balance Aging Report
**Priority:** High

As a collections officer, I want to run an aging report (0-30, 31-60, 61-90, 90+) so I can prioritize collection efforts.

**Acceptance Criteria:**
- Report shows counts and totals per aging bucket
- Filterable by area, date range, min balance
- Export to Excel with conditional formatting
- Scheduled delivery weekly via email
- Report caches for 1 hour; refresh button bypasses cache

### US-014: Per-Customer Aging Drill-Down
**Priority:** Med

As a collections officer, I want to click a customer to see their detailed aging with invoices in each bucket so I can take specific action.

**Acceptance Criteria:**
- Drill-down shows individual invoices grouped by aging bucket
- Each row: invoice#, date, due date, amount, days overdue
- Action column: Send Reminder, Add to Payment Plan, Flag Dispute
- Aging chart (bar) shows trend over last 6 months
- Download detail as PDF

## Payments (4 stories)

### US-015: Search Payments
**Priority:** High

As a cashier, I want to search payments by reference, customer, date range, amount, and status so I can find specific transactions quickly.

**Acceptance Criteria:**
- Search fields: Payment Ref (fuzzy), Customer (autocomplete), Date Range, Amount Range, Status (All/Pending/Completed/Reversed/Failed)
- Results table with columns: Ref#, Customer, Amount, Method, Date, Status
- Real-time search with debounce (400 ms)
- Results paginated and sortable
- Export filtered results to CSV

### US-016: Record New Payment
**Priority:** High

As a cashier, I want to record a payment against a customer account so I can apply funds received.

**Acceptance Criteria:**
- Payment form: Customer (autocomplete), Amount, Method (Cash/Cheque/Card/Bank Transfer), Reference#, Date, Notes
- Validation: amount > 0, customer exists, reference# unique
- Auto-allocation option to apply payment to oldest invoices first
- Upload receipt image (optional)
- Payment confirmation receipt generated on success

### US-017: Allocate Payment to Invoices
**Priority:** High

As a cashier, I want to manually allocate a payment across specific invoices so I can apply partial or directed payments.

**Acceptance Criteria:**
- List of unpaid/partially-paid invoices for the customer
- Input field per invoice for allocation amount
- Auto-calculated remaining unallocated balance
- Validation: total allocated ≤ payment amount
- Save creates PaymentAllocation records
- Invoice statuses update to Paid/Partially Paid

### US-018: Reverse Payment
**Priority:** Med

As a supervisor, I want to reverse a payment (with reason and approval) so I can correct erroneous transactions.

**Acceptance Criteria:**
- Reverse button on payment detail page
- Reason required (dropdown: NSF, Duplicate, Customer Request, Other)
- Supervisor approval required (second user credentials)
- Reversal creates offsetting journal entries
- Original payment status set to Reversed; allocations voided
- Audit trail links reversal to original payment

## Invoices (4 stories)

### US-019: Invoice List
**Priority:** High

As a billing operator, I want to view a list of invoices filterable by status, customer, date range, and area so I can manage billing.

**Acceptance Criteria:**
- Columns: Invoice#, Customer, Date, Due Date, Amount, Balance, Status (Draft/Issued/Paid/Overdue/Cancelled)
- Filters: Status (multi-select), Customer, Date Range, Area
- Bulk actions: Download Selected, Email Selected, Cancel Selected
- Sortable by any column
- Pagination with page size selector

### US-020: Generate Invoices
**Priority:** High

As a billing operator, I want to generate invoices in batch for a billing period so I can process monthly billing.

**Acceptance Criteria:**
- Wizard: Select Area → Billing Period → Preview → Confirm
- Preview shows customer count, total amount, potential errors
- Generation runs as background job with progress indicator
- Success summary: count issued, count failed, total value
- Failed items listed with error details for correction
- Invoice numbers auto-generated per sequence per area

### US-021: Adjust Invoice / Credit Note
**Priority:** High

As a billing operator, I want to adjust an issued invoice (credit note or debit note) so I can correct billing errors without deleting the original.

**Acceptance Criteria:**
- Adjustment types: Credit Note (reduce), Debit Note (increase)
- Reason required (Tariff Change, Meter Error, Manual Adjustment)
- Original invoice remains unchanged; adjustment references it
- New adjustment document created with sequential number
- Net balance reflects both original and adjustment
- Approval required for adjustments > 10,000 EGP

### US-022: Download Invoice PDF
**Priority:** Med

As a customer, I want to download my invoice as a PDF so I can file it for my records.

**Acceptance Criteria:**
- PDF template includes: company logo, customer info, meter details, charges table, VAT breakdown, QR code for payment
- Download button on invoice detail page
- Batch download from invoice list (max 50 at once)
- PDFs digitally signed with RSA-2048 (visible signature line on PDF)
- Generated PDF stored in blob storage; served via signed URL (24h expiry)

## Readings (3 stories)

### US-023: Unified Meter Readings List
**Priority:** High

As a meter reader, I want to view a unified list of all meter readings across areas so I can see the full picture.

**Acceptance Criteria:**
- Columns: Reading ID, Meter #, Customer, Area, Reading Value, Unit, Date, Source (Manual/AMI/Import), Status
- Filters: Status (Pending/Verified/Rejected), Date Range, Area, Meter Type
- Bulk approve/reject with reason
- Duplicate reading detection (same meter + same date → warning)
- Reading history graph per meter on detail expand

### US-024: Reading Review Queue
**Priority:** High

As a validation officer, I want a queue of readings flagged for review so I can approve or reject outliers before billing.

**Acceptance Criteria:**
- Queue sorted by priority (high deviation first)
- Each item shows: current reading, previous reading, consumption, % deviation
- Accept/Reject buttons; reject requires comment
- Bulk Accept for readings under threshold
- Rejected readings trigger re-read work order
- SLA timer: readings must be reviewed within 48 hours

### US-025: Smart Reading Validation
**Priority:** Med

As a validation officer, I want automatic validation rules on submitted readings so anomalies are flagged immediately.

**Acceptance Criteria:**
- Rules: Zero consumption (warning), Negative consumption (reject), > 300% vs. historical avg (flag), > meter max capacity (reject)
- Rules configurable per meter type via admin
- Violations create a ReadingReview record
- Validation runs synchronously on manual entry, async on AMI import
- Validation results visible in reading detail

## Tariffs (2 stories)

### US-026: Unified Tariff List
**Priority:** High

As a billing manager, I want to view all tariffs in a unified list so I can manage pricing across areas.

**Acceptance Criteria:**
- Columns: Tariff Name, Type (Water/Electric/Gas/Steam/Chilled Water), Version, Status (Draft/Active/Archived), Areas Applied
- Filters: Type, Status, Area
- Click opens tariff detail with charge breakdown
- Clone tariff function for creating variants
- Effective date range displayed per tariff

### US-027: Tariff Versioning
**Priority:** High

As a billing manager, I want to version tariff changes so I can modify rates without affecting current billing until the effective date.

**Acceptance Criteria:**
- New version created from current active version (copy)
- Version fields: version number (auto), effective date, status
- Only one version can be Active at a time per tariff
- Active version used for invoice generation for billing periods starting on/after effective date
- Version history viewable with diff highlighting
- Approval workflow: Draft → Pending Approval → Active

## Workspace (2 stories)

### US-028: Alerts Feed
**Priority:** High

As a billing operator, I want a real-time alerts feed on my workspace so I never miss critical system events.

**Acceptance Criteria:**
- Alerts displayed in reverse-chronological list
- Types: Payment Failed, Reading Anomaly, Invoice Error, System Warning, Approval Needed
- Each alert: icon, title, description, timestamp, link to relevant page
- Unread count badge in navigation
- Mark as read / Mark all as read
- Real-time via WebSocket; fallback to polling every 30s
- Configurable notification rules per user

### US-029: Ticket Kanban Board
**Priority:** Med

As a support agent, I want a kanban board for trouble tickets so I can manage support workflow visually.

**Acceptance Criteria:**
- Columns: Open, In Progress, Pending Customer, Resolved, Closed
- Cards show: ticket#, customer, subject, priority, assignee, age
- Drag-and-drop between columns updates ticket status
- Click card opens ticket detail with conversation thread
- WIP limits configurable per column
- SLA timer displayed on card (breached tickets highlighted red)

## Reports (2 stories)

### US-030: Run Report
**Priority:** Med

As a reporting analyst, I want to run reports from a catalog with parameters so I can generate custom analyses.

**Acceptance Criteria:**
- Report catalog grouped by category (Financial, Operational, Technical, Compliance)
- Each report has: name, description, parameter form
- Parameter types: date range, dropdown, checkbox, text
- Report runs as async job; progress bar shown
- Preview first 100 rows before full export
- Scheduled execution option (daily/weekly/monthly)

### US-031: Export Report
**Priority:** Low

As a reporting analyst, I want to export reports in multiple formats so I can share results with stakeholders.

**Acceptance Criteria:**
- Export formats: XLSX, CSV, PDF, HTML
- XLSX includes formatted headers, auto-column-width, frozen header row
- PDF includes page numbers and report title
- Email-to-stakeholder option with attached export
- Export history stored for 90 days
- Large exports (> 100k rows) streamed as chunks

## Admin (2 stories)

### US-032: User Management
**Priority:** High

As an admin, I want to manage users (create, edit, deactivate, assign roles) so I can control system access.

**Acceptance Criteria:**
- User list with columns: Name, Email, Role, Area(s), Status, Last Login
- Create user form: Name, Email, Password (auto-generated with option to email), Role, Area Assignment
- Edit user: change role, area, status; cannot edit password
- Deactivate: confirmation dialog, revokes active sessions
- Role change audit logged
- Bulk import users from CSV

### US-033: System Configuration
**Priority:** Med

As an admin, I want to view and edit system configuration parameters so I can tune the application.

**Acceptance Criteria:**
- Configuration grouped by module: Billing, Payments, Meters, Notifications, Security
- Parameters: key-value pairs with description
- Validation rules per parameter type (int, decimal, string, boolean, JSON)
- Change history logged with previous and new values
- Some parameters require restart notice
- Export/import configuration as JSON backup

## Locations (1 story)

### US-034: Location Hierarchy Tree
**Priority:** Low

As a billing operator, I want to browse a hierarchical tree of locations (Project → Building → Floor → Unit) so I can navigate the physical property structure.

**Acceptance Criteria:**
- Tree view with expand/collapse nodes
- Each node shows type icon and name
- Clicking a leaf node (Unit) shows associated customer and meter
- Search box filters tree nodes by name
- Breadcrumb trail shows current selection path
- Max depth: 5 levels
