## Attempt 7: Solar-Specific Fee from legacy code

The legacy `download_solar` code reads fees from invoice notes:
```python
admin_fee = 0; service_fee = 11
if 'رسوم ادارية' in p: admin_fee = float(p.split(':')[1].strip())
if 'رسوم خدمة' in p: service_fee = float(p.split(':')[1].strip())
```

The default is `service_fee = 11`. The historical invoices have `notes=''` (empty), meaning neither `admin_fee` nor `service_fee` was populated. The 36.10 and 9.10 values were generated by a **different version** of the code that used `service_fee = 36.10` (old) or `service_fee = 9.10` (post-2022 update).

## Conclusion

The 36.10 EGP minimum charge **cannot be decomposed using any known fee template** in the repository. It was produced by a version of the solar billing code where the `service_fee` was hardcoded to 36.10 EGP. This was a deliberate business rule in the legacy system, not derived from tariff tier rates or fee formulas.

The 9.10 EGP minimum (post-Feb 2022) was produced by an intermediate code version with `service_fee = 9.10`. The current Meter Verse code (`service_fee = 11`) is a THIRD value, suggesting the service fee was adjusted at least twice.
