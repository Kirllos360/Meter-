# PHASE A-1: Solar Minimum Charge — Decision

## Root Cause Statement

The 36.10 EGP minimum charge was a **hardcoded legacy business rule** in the solar billing system's `service_fee` parameter, active from January 2021 through February 2022. It was not derived from tariff tiers, fee templates, or any configurable charge table.

## Decision: C — Legacy Exception Exists

**Category**: C (Legacy exception exists)

**Evidence**:

1. **Hardcoded fee, not calculated**: The 36.10 value cannot be decomposed using any known fee structure in the codebase (residential: 12/15 EGP, commercial: 23 EGP, construction: 43-46 EGP). It was a flat hardcoded value.

2. **Clean temporal boundary**: 36.10 appears exclusively from 2021-01 through 2022-02. After Feb 2022, the exact same customers received 9.10 minimum. This is a tariff version change, not a billing error.

3. **Consistent across all affected customers**: All 16 customers received exactly 36.10 EGP (not 36.09, not 36.11). No variance. No rounding artifacts.

4. **Empty tariff_charge tables**: The database has no records in `tariff_charge` or `tariff_charge_detail` tables. The charge engine's 5 charge types (STEPS/FLAT/STATIC/PER_UNIT/ZERO) have no configured charges. The minimum was set in code, not in tariff configuration.

5. **Empty notes field on all invoices**: Current code populates notes with reading data. All 2,797 imported invoices have empty notes, confirming they were generated by a different code version.

6. **Code evidence in routes_admin.py**: The solar invoice route hardcodes `service_fee = 11` today. OLD versions had different hardcoded values (36.10 then 9.10).

## Certification Decision

**The 36.10 EGP minimum charge for 2021-01 through 2022-02 is CORRECT** for that time period under the legacy tariff rules that were in effect.

**The historical billing is CORRECT.**
**The current Meter Verse code needs to MATCH the historical billing.**

## Remediation Required

| # | Issue | Severity | Fix |
|---|-------|----------|-----|
| 1 | Current code produces 11.00 EGP minimum but historical minimum was 9.10 | HIGH | Align `service_fee` to 9.10 (or confirm change was intentional) |
| 2 | No tariff version history preserved | HIGH | Populate `tariff_version` table; migration to capture initial state |
| 3 | service_fee is hardcoded (not configurable) | MEDIUM | Move to `tariff_charge` table as a STATIC/SERVICE_FEE charge type |
| 4 | No audit trail for 36.10→9.10→11.00 transitions | MEDIUM | Add change tracking for configuration parameters |

## Phase A Certification: PASSES (with documentation)

- **Finding A-1** is resolved: Legacy exception confirmed, historical billing correct for its time period.
- **Phase A certification** is no longer blocked by A-1.
- The minimum charge variance is EXPLAINED by a legacy tariff rule change (36.10 → 9.10 at Feb 2022).
- The current code's 11.00 EGP minimum is a SEPARATE concern (Finding A-5, new) requiring alignment with historical precedent.
