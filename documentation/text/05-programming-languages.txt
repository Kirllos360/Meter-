Language Version Strategy

  Component  |  Strategy  

  Frontend TypeScript  |  Uses tsconfig.json in Frontend/ — strict mode, ES2017 target, bundler module resolution  
  Backend TypeScript  |  Uses tsconfig.json in backend/ — strict mode, ES2022 target, commonjs module (NestJS standard)  
  Node.js  |  Engine pinned to >=20 in backend package.json — matches frontend's Node version requirements  

Why TypeScript Everywhere?

  Reason  |  Detail  

  Type safety  |  Catches assignment/billing/ledger errors at compile time — critical for financial data  
  Shared types  |  Frontend types.ts enums must match backend Prisma schema enums  
  NestJS native  |  NestJS is built for TypeScript with decorator support  
  Next.js native  |  Next.js v16 has first-class TypeScript support  
  Graphify support  |  Graphify parses TypeScript AST for knowledge graph construction
